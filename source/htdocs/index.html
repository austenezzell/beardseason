Uno Grid

Usage

Uno requires a few classes. The .wrap class wraps all elements. It is horizontally centered and has a max-width. Columns are wrapped in a .row class, and each column requires a .col or .col-center class. Use the .span- classes to span the columns by n across different breakpoints. The default number of columns is 12, seen above. Width, columns, and gutter can be customized in _variables.scss.

It should be noted that column widths start at mobile and scale to desktop. For example the "md" breakpoint will override the "xs" breakpoint on medium-sized screens.

Each tier of classes scales up from "xs", meaning if you plan on setting the same widths for "lg" and "md", you only need to specify "md". "lg", "md", and "sm" grids automatically stack at "xs" width.

Mixing breakpoints

The Uno grid system has four tiers of classes: xs (phones), sm (tablets), md (small desktop), and lg (full-width). You can combine these classes to create more flexible layouts.

.span-xs-12 .span-lg-8
.span-xs-6 .span-lg-4
.span-xs-4 .span-lg-5
.span-xs-4 .span-lg-2
.span-xs-4 .span-lg-5
.span-xs-6 span-sm-4 span-md-8 span-lg-6
.span-xs-6 span-sm-8 span-md-4 span-lg-6

Offset

Offset n-columns left and reset at specific breakpoints. Just subtract 1 column from the element you add the offset to, and be considerate of siblings. Offsets larger than "xs" automatically reset at "xs" width.

.span-sm-5 .span-xs-5 .offset-lg-1 .offset-sm-1
.span-sm-5 .span-xs-5 .offset-lg-1 .offset-sm-1
.span-sm-5 .span-xs-5 .offset-lg-1 .offset-sm-1
.span-sm-5 .span-xs-5 .offset-lg-1 .offset-sm-1
.span-xs-10 .offset-xs-2 .span-sm-12 .offset-sm-0 .span-md-6 .offset-md-6

Push / Pull

By default, all coulmns are floated left.However, on some breakpoints you may want an element on the the right and others on the left.

Using the .push and .pull class reorder you columns. Be advised, using this class can have adverse effects on the order of your elements, and weird stuff can happen if you're not careful.

[first in DOM] .span-xs-8 .push-lg-4
[second in DOM] .span-xs-4 .pull-lg-4

Nesting is possible and you can do some really weird stuff.

.span-xs-8 .push-lg-2
[nested and pulled] .span-xs-6 .pull-xs-1
[nested and pushed] .span-xs-6 .push-xs-1

Top, Middle, Bottom & Start, Center, End

Adding these classes to the .row element allows flexbox to work its magic.

.top-, .middle, and .bottom- refer to the alignment of all columns inside the container.

.start-, .center-, and .end- refer to the content inside the columns. It's a bit like text-align but works on blocks.

They can be mixed and matched in lieu of the push/pull classes to align content.

.top-xs .start-xs
.top-xs .start-xs
.middle-xs .center-xs
.middle-xs .center-xs
.bottom-xs .end-xs
.bottom-xs .end-xs

Adding .top-, .middle, and .bottom- to a .col element applies the alignment individually for more dynamic layouts. These override the class on .row, so be careful.

.top-xs
.middle-xs
.bottom-xs
